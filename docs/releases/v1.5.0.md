# Nexa WebApp v1.5.0 - Backend IAM Integration

## Overview
This release integrates the core authentication flows with the real .NET backend and aligns workspace access with the seeded backend profiles.

## Main Changes

### IAM / Authentication Integration
- **Real Backend Authentication:** Configured `verifyCredentials` in `iam.application.js` to POST to the C# authentication API when `VITE_CORE_BACKEND_ENABLED` is `true`.
- **Quick Access Alignments:** Updated user profiles to match the seeded backend credentials.
- **Session Persistence:** Configured Pinia and `localStorage` to preserve user profile information (`username`, `role`, `clientId`, `accessToken`) across browser page refreshes, resolving broken layouts on reload.
- **Buyer Client Mapping:** Implemented a role-based mapping so backend buyer accounts link to `clientId: "CLI-001"` when the client profile endpoint is not available.

### Sales Boundary
- **B2B Client Boundary:** Kept client management dependent on backend support instead of replacing the missing endpoint with local data.
- **Clean Assets:** Removed deprecated duplicate logo assets (`logo.svg` and `logo-white.svg`) to keep the source tree clean.

### Documentation
- Updated `README.md` to document the steps to configure and run the WebApp connected to the C# backend.
