# Nexa WebApp v1.5.0 - Backend IAM Integration & Client Fallback Resilience

## Overview
This release integrates the core authentication flows with the real .NET backend and implements resilient local fallbacks to ensure the application remains stable and usable when backend services are partially available.

## Main Changes

### IAM / Authentication Integration
- **Real Backend Authentication:** Configured `verifyCredentials` in `iam.application.js` to POST to the C# authentication API when `VITE_CORE_BACKEND_ENABLED` is `true`.
- **Quick Access Alignments:** Updated user profiles to match the seeded backend credentials.
- **Session Persistence:** Configured Pinia and `localStorage` to preserve user profile information (`username`, `role`, `clientId`, `accessToken`) across browser page refreshes, resolving broken layouts on reload.
- **Buyer Client Mapping:** Implemented a safety mapping to guarantee that logging in as `buyer.demo@nexa.com` links correctly to `clientId: "CLI-001"` (Importaciones y Comercio Internacional S.A.).

### Sales & Resilient Fallbacks
- **B2B Clients Fallback:** Added a local fallback client list in `data.store.js` and updated `clients-api.js` to intercept connections failures and return these mock client accounts, ensuring the Sales and Buyer dashboards load correctly even without an active client API.
- **Clean Assets:** Removed deprecated duplicate logo assets (`logo.svg` and `logo-white.svg`) to keep the source tree clean.

### Documentation
- Updated `README.md` to document the steps to configure and run the WebApp connected to the C# backend.
