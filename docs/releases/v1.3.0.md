# v1.3.0

Frontend product readiness sweep after the bounded-context refactor.

## Scope

- Fixed case-sensitive entity and repository filenames to match kebab-case imports in Linux CI and production hosts.
- Replaced outdated prototype API wording with temporary Mock API language.
- Improved S3 Buyer Portal desktop width, footer behavior, mobile bottom navigation, and hamburger menu access.
- Improved S1/S3 catalog parity with brand, cold type, stock, and compact responsive filters.
- Expanded i18n coverage for Buyer Portal home, catalog filters/actions, Commercial Dashboard, shared status labels, cart controls, and Nexa Assistant.
- Added a backend-ready buyer assistant API boundary at `/api/v1/buyer-assistant/messages` with local fallback behavior until the real backend exists.

## Validation

- `npm run build` passed locally.
- Grep checks passed for old feature-root imports, uppercase entity/repository paths, and outdated prototype API wording.
- Browser QA should focus on `/portal/home`, `/portal/product-catalog`, `/ops/commercial`, and `/ops/commercial/product-catalog` across desktop and mobile viewports.
