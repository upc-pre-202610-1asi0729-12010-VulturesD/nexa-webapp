# Nexa WebApp v1.8.0 - Mock API Cleanup & Layout Polish

## Summary

This minor release removes the legacy mock API (JSON Server) dependency and introduces dynamic in-memory dataset cloning to support local-only contexts without triggering external HTTP requests. It also polishes layout responsiveness across sales and operations dashboards.

## Changes

### API & Store Cleanup
- Removed `json-server` from devDependencies and deleted local fake API runner tasks.
- Cleaned all mock-related parameters (`useMockApi`, mock fallbacks) from Axios base clients and endpoints.
- Populated global Pinia store with local static initial dataset (`src/shared/data/initial-data.json`) cloned at bootstrap to support offline resources.
- Cleared academic demo logins, replacing them with standardized final user accounts (`buyer@nexa.com`).

### Layout & Style Polish
- Fixed catalog table responsiveness in `create-order-view.vue` to avoid item clipping on smaller viewports.
- Restructured quick-actions grid in `ops.css` using fluid auto-fit columns.

## Validation

- Verified build compilation passes without warnings (`npm run build`).
- Checked Axios endpoints connection mapping to live platform backend.
- Validated in-memory store mutations.
